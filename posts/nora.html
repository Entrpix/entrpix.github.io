Nora, AUR on all Distros

If you somehow don't know, the AUR is a repository of packages for Arch Linux maintained by the community instead of a dedicated team.

It's one of the main reason people love Arch so much, if you need a package, the AUR has it.

Theres one problem though, the AUR is only available on Arch and other Arch-Based distros, Nora aims to fix that.

Nora is a CLI tool that uses Distrobox to install packages from the AUR on any distro.

Distrobox is another CLI tool using Docker that allows you to test any distro on well, any distro.

Nora uses an Arch Distrobox instance that acts as a easy-to-interact-with VM for the AUR.

Nora uses Yay as the AUR helper.

Why can't we just install Yay on our distro? Good question, while you can install Pacman, Arch's package manager and makepkg on a distro other then Arch, its well, painful.

Once you setup a basic Arch container with Yay in Distrobox you can run nora init to setup Nora.

From their you can use Nora to install AUR packages on any distro.

Their are still some issues with this though, for example, once you install a package you will probably want to appear in your distro's application menu. You will need to run nora export

I actually have no clue if this works as I'm using WSL and I don't have a GUI, but it should work.

If that doesn't work Nora also has nora open which can open packages if they dont show up, still annoying though.
