Honey (Pollen, but Windows)

If you have seen Pollen you know that it's a tool for testing and overwriting Google Enterprise Policies.

However, this tool is only for ChromeOS devices.

If you don't know how Pollen works, I'll give you a quick rundown.

In ChromeOS it stores User Enrollment Policies locally, similar to how you would set them in Linux.

Howver you can only set user policies, not device policies. Devices policies are any policies starting with "Device"

In Linux you can set ChromeOS policies by editing a JSON file stored at /etc/opt/chrome/policies/managed/managed_policies.json

Pollen works in the exact same way, policies are fetched from the Cloud and then written to this file.

This means if your ChromeOS device has Devmode enabled, you can edit these policies.

However, this only works on ChromeOS and Linux. What about Windows?

That's where the Windows Registry comes in. If you have Chrome installed policies are stored at Computer\HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Google\Chrome\

Referring to the Policies list (provided above), you can edit the Registry to change the policies!

For example if you create the key ExtensionInstallBlocklist (under the policies path in the Registry) and create a string named 1 and set the value to be bmnlcjabgnpnenekpadlanbbkooimhnj It will block the Chrome Extension, Honey.

To see this in effect, open Chrome and go to chrome://policy and click "Reload Policies" you will notice a policy called "ExtensionInstallBlocklist" was created with the value ["bmnlcjabgnpnenekpadlanbbkooimhnj"]

And sure enough, if you go to Honey's extension page, it says "Your admin has blocked this item".
